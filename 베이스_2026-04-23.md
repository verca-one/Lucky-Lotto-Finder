# 복권명당 (Lucky Lotto Finder) - 진행상황 저장
**날짜**: 2026-04-23 (작업 완료)

## ✅ 완료된 작업

### 1. 크롤링 시스템 구축
- **파일**: `data/crawling/lotto_crawler.py`
- **기능**:
  - 동행복권 API에서 당첨 판매점 정보 크롤링
  - 로또(L645), 연금복권(P720), 스피또(S505) 3가지 종류 수집
  - 각 회차별 당첨번호(`numbers` 필드) 저장
  - 자동/수동/반자동 판매 방식(`method` 필드) 저장
  - 최신 회차부터 역순 크롤링 (빠른 데이터 수집)
  - 10회차마다 중간저장 (중단 시 재시작 가능)
  - `--type lotto|pension|speeto` 옵션으로 개별 크롤링 가능

### 2. 자동화 스케줄러 구축
- **파일**: `data/crawling/lottery_scheduler.py`
- **기능**:
  - Firebase Realtime 초기화 및 데이터 업로드
  - 크롤링 결과를 Firestore `win_history` 컬렉션에 저장
  - 자동 스케줄링: schedule 라이브러리 사용

### 3. 자동 크롤링 일정
```
🔵 매주 목요일 19:30 → 연금복권 크롤링 + Firebase 업로드
🔴 매주 토요일 09:30 → 로또 크롤링 + Firebase 업로드
⭐ 매일 00:00 → 스피또 크롤링 + Firebase 업로드
```

### 4. Firebase 설정
- **파일**: `data/crawling/serviceAccountKey.json`
- **상태**: ✅ 초기화 완료
- **저장 위치**: Firestore `win_history` 컬렉션
- **무료 플랜**: 쓰기 1,046/일 (무료 한계 20,000/일의 5.2%) → 충분!
